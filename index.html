Here comes the batch.

  • First we are going to talk about how to determine the number of strings in a text file. To accomplish this we are going to use the power of the find command. What we are doing here is looking for empty strings in a text file and only use the strings that don't contain them (/v switch). Every string in a file contains something. It can be an array of symbols or something that you cannnot even see, like <CR> (carrige return), <LF> (line feed) and other non-printable characters. So until you hit the End Of File (EOF) pointer every line will have something inside of it. In other words we seek for the lines with content and you know what, we're lucky because every line in a file has it!

    find /c /v "" [file name]

    /c switch does the counting.

  • Another very common task is obtaining date and time. There are a lot of different methods to do so, but I find the one that uses WMI (Windows Management Instrumentation) to be the most reliable out there. The advantage of this approach is that regional settings won't affect the output.

    @echo off
    setlocal enableDelayedExpansion
    for /f "tokens=2 delims==+-" %%i in ('WMIC OS Get LocalDateTime /value ^| findstr "="') do (
    set _Date=%%i
    set YYYY=_Date:~0,4!
    set MM=!_Date:~4,2!
    set DD=!_Date:~6,2!
    set HH=!_Date:~8,2!
    set Min=!_Date:~10,2!
    set Sec=!_Date:~12,2!
    set Mili=!_Date:~-6!
    )
